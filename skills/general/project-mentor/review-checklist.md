# 导师检查清单

## A. 开始前

- 当前问题属于哪一类：
  - 学习理解
  - 功能规划
  - 代码修改
  - bug 调试
  - 架构设计
  - 产品取舍
- 现在做这件事时机对吗？
- 范围是不是太大？
- 需不需要先冻结加功能？

## B. 本轮边界

- 本轮目标是否明确？
- 最小闭环是否明确？
- 验收标准是否明确？
- 本轮不做什么是否明确？
- 哪些文件可能相关？
- 哪些地方不要乱动？

## C. 项目理解

- 有没有先看项目结构？
- 有没有先看入口文件和主流程？
- 有没有区分完成 / 半完成 / 只有 UI / 状态不明？
- 有没有识别出高风险区域？

## D. 教学质量

- 有没有先讲当前所需，不一次性灌输整套理论？
- 有没有把知识点映射回真实项目文件？
- 有没有使用：
  - `【概念】`
  - `【重点】`
  - `【代码】`
  - `【误区】`
  - `【自测】`
  - `【笔记】`
- 有没有给用户留自测题？

## E. 改代码前

- 有没有先说明为什么改？
- 有没有说明改哪些文件？
- 有没有说明哪些地方不改？
- 有没有说清风险？
- 有没有说清怎么验收？
- 方案是不是最小可行修改？

## F. 结束前

- 用户今天学到了什么是否说清楚？
- 项目推进了什么是否说清楚？
- 下一轮先学什么是否说清楚？
- 是否需要进入下一阶段，还是继续留在当前阶段？
