# project-mentor 示例

## 第一次进入项目时

```text
用 project-mentor 带我做这个项目的第一轮项目考古。先冻结新增功能，先别急着写代码，先帮我看懂这个项目已经有什么、主流程是什么、现在最该先理解哪一块。
```

## 我已经做不下去的时候

```text
我在这个项目里已经越做越乱了。不要先帮我加功能，先像一个严谨导师一样判断我现在到底该理解什么，不该碰什么。
```

## 做需求前先判断

```text
先别写代码。用 project-mentor 先判断这个需求现在值不值得做，范围是不是太大，本轮最小闭环是什么，验收标准是什么。
```

## 先做项目地图

```text
不要直接改项目。先带我做页面地图、功能地图、文件地图和数据地图，再告诉我一个初学者应该按什么顺序理解这个项目。
```

## 想跳过理解直接开干时

```text
我现在有点想直接上手改代码了。你先拦我一下，判断我是不是在跳过理解，先告诉我我还缺哪几个前提。
```

## 调 bug 时

```text
不要直接给我修复答案。先带我学会怎么复现这个 bug、怎么缩小范围、怎么判断是哪一层的问题，再决定要不要改代码。
```

## 学 React / API / 状态管理时

```text
我没有基础。请你不要默认我懂这些概念，就围绕这个项目当前用到的部分，小步给我补 React、API、状态这些知识。
```

## 练架构判断时

```text
不要只解释这一行代码。带我看这个模块和别的模块是什么关系，这个状态为什么该放这里，这次改动会不会增加维护成本。
```

## 做完一轮后的复盘

```text
用 project-mentor 帮我复盘今天这轮：我学到了什么、项目推进了什么、我还卡在哪里、下一轮最应该先学什么。
```

## 适合你个人口吻的说法

```text
你现在先别顺着我，先像导师一样判断一下，我这个想法现在该不该做。如果不该做，就直接说不建议现在做，并告诉我我应该先补什么。
```

```text
先把这个项目当教材，不要先当任务。你带我先看懂，再决定要不要动代码。
```

```text
我想通过这个项目学会独立判断，不是只把功能堆出来。你用 project-mentor 的方式带我推进。
```
